## 見どころ

- **2テンプレート方式**(①設定マスタ10列 / ②従業員データ8列)の設計
- **順序式アップロード**(1st完了後に2nd有効化) vs **単独選択**(既存データありの場合)の並立

- **入力ガイドv2** で導入ユーザーの入力ミス防止

## 概要

ラクミータイムレコードの設定データ(従業員、雇用区分、店舗、業態・ブランド)を外部からインポートする機能の仕様書。

### インポート方式

| 方式 | 対象ユーザー | 特徴 |
|---|---|---|
| テンプレートインポート | 全ユーザー | ラクミー指定テンプレートにデータを入力してインポート |
| 外部勤怠サービスインポート | KoT/Jobcanユーザー | CSVエクスポートデータを列マッピングしてインポート |

---

## 1. テンプレートインポート

### 1.1 テンプレート構成（2ファイル）

| テンプレート | 列数 | 内容 | 1行の粒度 |
|---|---|---|---|
| **① 設定マスタ** | 10列 | 業態・ブランド + 店舗 + 雇用区分 | 1行 = 1店舗×1雇用区分の組み合わせ |
| **② 従業員データ** | 8列 | 従業員情報 | 1行 = 1従業員 |

### 1.2 ① 設定マスタ テンプレート列

| 列名 | 必須 | 備考 |
|---|---|---|
| 業態・ブランド名 | ★ | 内部で重複排除して登録 |
| 店舗名 | ★ | 内部で重複排除して登録 |
| 郵便番号 | ★ | ハイフン付き（例: 150-0002） |
| 住所 | ★ | |
| 雇用区分名 | ★ | 内部で重複排除して登録 |
| 雇用区分コード | | 空欄時は自動生成 |
| 給与体系 | ★ | 月給 / 日給 / 時給 |
| 金額 | ★ | 給与体系に対応する金額 |
| 月次契約労働時間 | | 単位: 時間 |
| 月みなし残業時間 | | 単位: 時間 |

### 1.3 ② 従業員データ テンプレート列

| 列名 | 必須 | 備考 |
|---|---|---|
| 従業員名 | ★ | |
| 電話番号 | ★ | ハイフン付き |
| メールアドレス | ★ | |
| 従業員コード | | 空欄時は自動生成 |
| 雇用区分名 | ★ | ①の雇用区分名と一致させる |
| 店舗名 | ★ | ①の店舗名と一致させる |
| 交通費単位 | ★ | 月額 / 日額 |
| 交通費 | ★ | 交通費単位に対応する金額 |

### 1.4 インポート順序

```
① 設定マスタ → ② 従業員データ
```

- 両方選択時は順序式アップロード（1stが完了後に2ndが有効化）
- 片方のみの選択も可能（例: 雇用区分が登録済みなら②のみ）
- 内部で自動仕分け: ①のデータから業態・ブランド / 店舗 / 雇用区分を重複排除して各テーブルに登録

### 1.5 画面フロー

```
Step 1: テンプレート選択 + ファイルアップロード
  ↓
Step 2: 確認・実行（仕分け結果プレビュー + 従業員プレビュー）
  ↓
Step 3: 完了
```

### 2.3 画面フロー

#### 通常フロー（マッピング設定済み）

```
Step 1: サービス選択 + データソース選択 + ファイルアップロード
  ↓
Step 2: プレビュー・確認（統合プレビュー + 未マップ項目表示）
  ↓
Step 3: 完了（補完テンプレートDL導線）
```

#### 初回フロー（マッピング未設定）

```
Step 1: サービス選択 + データソース選択 + ファイルアップロード
  ↓
初回マッピング設定（全データソースのマッピングを設定）
  ↓
Step 2: プレビュー・確認
  ↓
Step 3: 完了
```

### 2.4 アップロード方式

| データソース選択 | アップロード方式 |
|---|---|
| 1つのみ | シンプルな1口アップロード |
| 2つ選択 | 順序式アップロード（1st完了後に2ndが有効化） |

**アップロード順序:**
- KoT: 1st employee_info → 2nd hr_
- Jobcan: 1st staff-info- → 2nd jobcan_employees
- 間違ったファイルをアップした場合はエラーアラートで受付拒否

### 2.5 取り込み設定

データソース毎に独立管理:
- ファイル読み取り設定（文字コード / 区切り文字 / ヘッダー行 / データ開始行）
- 列マッピング設定
- 名前を付けて保存 → 設定履歴一覧で管理

### 2.6 マッピング設定

- **初回**: 初回マッピング設定画面で全データソースをまとめて設定
- **以降**: 取り込み設定画面の「マッピングを編集」リンクからデータソース個別に編集
- KoT/Jobcanはプリセットマッピングが自動適用される

## 3. マッピング不可項目の補完

### 3.1 補完が必要になるケース

| サービス | マッピング不可の必須項目 |
|---|---|
| KoT (employee_info) | 電話番号, メールアドレス, 交通費単位, 交通費 |
| KoT (employee_info + hr_) | 電話番号, メールアドレス, 交通費単位 |
| Jobcan (staff-info-) | 交通費単位 |
| Jobcan (staff-info- + jobcan_employees) | 交通費単位 |

### 3.2 補完方法（完了画面から導線提供）

#### 方法1: 一括補完（CSV取り込み）

1. 完了画面で「補完用テンプレートをダウンロード」
2. テンプレートは**マッピング結果に応じて動的に列生成**（従業員コード + 不足項目のみ）
3. ユーザーが不足データを入力
4. テンプレートインポートの「② 従業員データ」で重複時「上書き更新」を選んでインポート
5. 従業員コードをキーに不足項目だけ補完される

#### 方法2: 個別編集（従業員画面）

1. 完了画面で「従業員一覧（未入力フィルター）→」
2. 従業員一覧画面で未入力項目ありの従業員をフィルター表示
3. 各従業員の編集画面から個別に入力

### 3.3 プレビュー画面でのデフォルト値設定

マッピング不可項目のうち選択肢が限定されるもの（交通費単位: 月額/日額）は、プレビュー画面でデフォルト値を設定してインポート可能。

## 4. 内部仕分けロジック

### 4.1 テンプレートインポート時

① 設定マスタのデータから自動仕分け:

| 処理順 | 対象 | ロジック |
|---|---|---|
| 1 | 業態・ブランド | A列を重複排除して登録 |
| 2 | 店舗情報 | B列を重複排除、A列で業態紐付け |
| 3 | 雇用区分 | E列を重複排除、F〜J列で詳細設定 |
| 4 | 従業員 | K〜P列は全行登録、雇用区分名・店舗名で紐付け |

### 4.2 外部SaaSインポート時

2つのデータソース選択時の統合ロジック:
- 1st（設定データ）をベースにレコード作成
- 2nd（給与データ）を従業員コードをキーに補完・統合
- 2ndにしかない列（給与体系、通勤手当等）を1stのレコードにマージ

### 4.3 重複時の処理

| 設定 | 動作 |
|---|---|
| スキップ（既存を保持） | 同一キーのレコードが存在する場合、既存データを変更しない |
| 上書き更新 | 同一キーのレコードが存在する場合、新しいデータで上書き |

ユニークキー:
- 従業員: 従業員コード（システムID）
- 雇用区分: 雇用区分コード or 雇用区分名
- 店舗: 店舗名
- 業態・ブランド: 業態・ブランド名
