# CHANGELOG — タイムレコード import-spec

判断経緯＝`JUDGMENT_LOG.md`。全体の入口・正本は owner-mock（`rakmy-owner-registration-mock`）。

## 2026-07-13

### Changed（外部勤怠サービス取込 s-saas1 UI）
- **連携元サービス選択 → 取り込みステップへスライド**：サービス（KOT/JC/その他）を選ぶとピッカーが退場し、そのサービス専用の取り込みウィザードへスライド表示（`saasSyncSlideView` / `saasPickerConfirmed`）。「連携元サービスを変更」で選択へ戻れる。
- **選択中サービス名を左上に明示**：「取り込むデータを順番に」の上に `連携元サービス [ロゴ] サービス名` バーを表示（何に対するアップロードか明確化）。その他は取り込みステップの代わりに問い合わせ案内。
- **「取り込み設定」ボタンを「取り込むデータを順番に」カードの右上へ移設**（フッターの btn-group から移動）。

### Changed（テンプレ取込ウィザード s-tpl1）
- **確認画面への遷移ボタンの重複を解消**：ウィザードカード内の「確認画面へ →」とフッターの「確認画面へ →」が二重だったため、**フッター側（`tpl-footer-confirm`）を撤去**。確認導線はカード内 primary ＋ 完了モーダルに一本化。
- **管理者(任意)アップ完了モーダル(B)を追加**：必須4項目アップ完了モーダル(A)で「⑤ 管理者(任意)も取り込む」を選び、管理者テンプレDL→アップロードを終えると、**「必須＋任意データのアップロードが完了しました」モーダル**を表示し、その場の「→ このまま確認画面へ進む」で s-tpl2 へ遷移（`showTplAdminDoneModal` / `tplAdminDoneGoConfirm`）。

### Changed（完了系UIの整理）
- **アップロード完了モーダルと最終「インポート完了」画面の役割を明確化**：中間モーダル（必須アップ完了時）の見出しを「取り込みが完了」→**「アップロードが完了」**に変更し、「実際の取り込み（登録）は確認画面で実行し、完了画面に結果表示」と補記。→ 2つの完了表示（中間モーダル＝アップ完了の案内 / s-tpl3・s-saas4＝実行後の結果）が別物と分かるように。
- **「インポート完了」画面をモーダル幅→画面幅に拡大**：`.result-detail max-width:360px`→フル（`.result-center` を最大680px中央寄せ）、アイコン/見出し/行の文字サイズを拡大（56→64 / 18→24 / 13→14px）。s-tpl3・s-saas4 共通。

### Fixed
- **中断ラベルの文言是正**：一時保存ドラフトの `label` を「✓ 必須項目すべて完了(登録前準備完了)」→ **「必須ファイルをすべてアップ済み（本取り込み前）」**に。打刻記録の再開バナーで「完了」と誤読される問題を解消（owner-mock 側バナーと整合）。
- **「初期セットアップへ戻る」ボタンの遷移を確実化**（`bGoBackToSetupGuide`）：親タブ(初期セットアップ)から開かれた場合は `window.opener`（同一オリジン）で親を `#b0` に切り替えて前面化＋自タブを閉じる。単独タブ時はこの画面を `#b0` へ遷移。旧実装は `window.close()` 依存で単独タブ時に不発だった。
- **未アップロード時は「一時保存」「一時保存して中断」を押下不可に**（テンプレ／外部勤怠サービス両ウィザード s-tpl1 / s-saas1）。
  - 判定を新ウィザードのアップロード有無（`wizHasAnyUpload`）に一本化。`syncWizSaveButtons()` を `renderTplWizard()`/`renderSaasWizard()` 末尾で呼び、フッターの一時保存ボタン＋page-header の中断ボタンを同期。
  - 旧 `refreshSaveButtonsState` は legacy チェックボックス（ウィザードでは非表示）で判定しており、フッターだけ有効化される不整合があったのを是正。1件でもアップロードすると両ボタン有効化。
