# CHANGELOG — タイムレコード import-spec

タイムレコード CSV 取込フロー（テンプレート／外部勤怠サービス連携）モックと仕様の変更履歴。
判断経緯＝`JUDGMENT_LOG.md`。全体の入口・正本は owner-mock（`rakmy-owner-registration-mock`）。

---

## 2026-07-16

### Fixed（単体取込画面の「閉じる」戻り先を入場経路で出し分け）
- **入場経路で戻り先を分岐**（`singleClose`/`_singleEntryFrom`）：インポートTOP（運用ハブ）経由なら**TOP（`s-top`）へ戻る**、設定画面の「📥 CSVインポート」経由（`?type=&mode=`）なら**元の設定画面へ戻る**（`returnToSourceSettingByKey`）。従来は常に owner-mock 設定画面へ飛び、ハブ経由でも違う画面へ遷移していた不具合を是正。

### Changed（単体取込画面の作業ステップ表記を撤去＝旧仕様の整理）
- **番号付きプログレスバー（連携元→アップロード→プレビュー→完了）を撤去**：運用の単体取込は連携元・取込対象が確定済みで実作業はアップロードのみのため、連番の作業ステップ表示をやめた。
- 各カード見出しの**連番（1/2/3）も削除**（「取込対象」「ファイルをアップロード」）。連携元は素の1行「設定済み：{サービス名}」。

### Changed（インポートTOP＝運用ハブに刷新・入口分離）
- **左ナビ「インポート」＝運用インポートのハブ**（`s-top`）に。種別（業態/店舗/雇用区分/従業員/管理者）ごとの単体取込入口。
- **初期セットアップの一括取込（`s-tpl1`/`s-saas1`）はTOPから撤去**し、オンボーディング b0（`#tpl-setup`/`#saas-setup`）専用に限定。初期（一括）と運用（単体）の入口・表示を明確化。
- **インポート方式（テンプレ/外部勤怠）は初期設定で決定済みのグローバル設定**：運用では毎回選ばせず、現在方式を上部ステータス表示。種別「取り込み →」は現在方式で直接単体取込へ（`hubGoEntity`→`s-{tpl|saas}-single-X`）。
- **右上「⚙ インポート方式を再設定」**：方式を選び直すスライド（`hubReconfig`/`hubSetMethod`）。テンプレ/外部勤怠の2カード、現在方式に「現在の設定」バッジ、`localStorage` 保持。
- **外部勤怠選択時は連携元サービス選択スライドを1枚追加**（キングオブタイム/ジョブカン/その他・`hubSetService`／初期セットアップと同方式）。現在方式は**「🔄 外部勤怠サービス: {サービス名}」**と連携元まで明示。選び終えると種別一覧へ、種別クリックで `s-saas-single-X` へ（`_activeSaasSource` 同期／`jc`→`jobcan` マッピング）。
- **単体取込画面（s-saas-single-X）の連携元「再選択」を廃止**：ハブで選択済みのため、Step1 を選択UI→**読み取り専用表示「連携元サービス（設定済み）: {サービス名}」**に（`applyToSingleScreen` で `saas-src-cards` を隠し `saas-src-ro` を表示）。変更はTOPの「方式を再設定」から。二重の連携元選択を解消。

## 2026-07-13

### Changed（外部勤怠サービス取込 s-saas1 UI）
- **連携元サービス選択 → 取り込みステップへスライド**：サービス（KOT/JC/その他）を選ぶとピッカーが退場し、そのサービス専用の取り込みウィザードへスライド表示（`saasSyncSlideView` / `saasPickerConfirmed`）。「連携元サービスを変更」で選択へ戻れる。
- **選択中サービス名を左上に明示**：「取り込むデータを順番に」の上に `連携元サービス [ロゴ] サービス名` バーを表示（何に対するアップロードか明確化）。その他は取り込みステップの代わりに問い合わせ案内。
- **「取り込み設定」ボタンを「取り込むデータを順番に」カードの右上へ移設**（フッターの btn-group から移動）。

### Changed（外部勤怠取込 s-saas1 — 完了スライド）※2026-07-16
- **全アップロード完了時はスライドを「完了」表示に畳む**：個別のDL/アップUIの代わりに「✅ すべてのデータをアップロード完了」バナーを表示（取り込み対象一覧・戻るは維持）。**確認画面への遷移は下部フッターの「確認画面へ →」のみ**（スライド内には確認ボタンを置かない＝PdM選択）。冗長な下部ヒントは撤去。

### Changed（外部勤怠取込 s-saas1 / 取り込み設定）
- **SaaSの完了モーダルを廃止**：外部勤怠は必須/任意が混在するため、テンプレ取込のような「必須完了モーダル」は出さない（トリガー・デモボタン・モーダル本体を撤去）。
- **「取り込み対象」サマリを追加**（アップロード欄の下）：**アップ済み（=取り込み対象）だけを育てて明示**する方式。全項目を常時 `○` で並べる案は上部ステップドットと重複するため不採用。
  - **視覚的に“従”へ調整**：当ステップの「アップロード」操作より目立たないよう、緑ヘッダー＋警告バーの主張の強い箱をやめ、控えめなグレーの小サマリに。**0件時は1行に圧縮**（必須未アップのみ小さく赤字）。アップ済みは `✓ 項目名 … ファイル名` を淡色で列挙、残りの任意は件数のみ案内。
- **ステップ2の見出しを「アップロード」に簡素化**（旧「記入してアップロード」「エクスポートしてアップロード」）。テンプレ取込 s-tpl1 も同様。手順詳細は各項目の hint に集約。
- **「マッピングを編集」をテキストリンク→明示ボタン化**（取り込み設定 s-saas-settings の各データソース）。枠付き青ボタン＋✏️アイコンで視認性向上。

### Changed（両ウィザードの「次へ / 確認画面へ」の整理 — s-tpl1 / s-saas1）
- **確認画面(プレビュー)への遷移をフッターの1ボタンに一本化**：カード内の「次へ」は“次のアップロード項目へ進む”用途のみに限定。**全アップロード完了時（または最終項目）はカード内の「次へ」を非表示**（戻る・ステップ直接クリックは可能）。確認画面へは**枠外フッターの「確認画面へ →」（tpl）/「次へ: プレビュー →」（saas）のみ**で遷移。フッターボタンは**必須アップ完了で活性化**。
  - ※ 以前「フッター側を撤去（カードに一本化）」としたが、PdM 指示により**逆（フッターに一本化・カードの確認ボタン撤去）**へ最終決定。
  - 全完了時はカード下部に「✓ すべてアップロード済み — 下部の『確認画面へ →』で進めます」を表示。

### Changed（テンプレ取込ウィザード s-tpl1）
- **管理者(任意)アップ完了モーダル(B)を追加**：必須4項目アップ完了モーダル(A)で「⑤ 管理者(任意)も取り込む」を選び、管理者テンプレDL→アップロードを終えると、**「必須＋任意データのアップロードが完了しました」モーダル**を表示し、その場の「→ このまま確認画面へ進む」で s-tpl2 へ遷移（`showTplAdminDoneModal` / `tplAdminDoneGoConfirm`）。

### Changed（完了系UIの整理）
- **アップロード完了モーダルと最終「インポート完了」画面の役割を明確化**：中間モーダル（必須アップ完了時）の見出しを「取り込みが完了」→**「アップロードが完了」**に変更し、「実際の取り込み（登録）は確認画面で実行し、完了画面に結果表示」と補記。→ 2つの完了表示（中間モーダル＝アップ完了の案内 / s-tpl3・s-saas4＝実行後の結果）が別物と分かるように。
- **「インポート完了」画面をモーダル幅→画面幅に拡大**：`.result-detail max-width:360px`→フル（`.result-center` を最大680px中央寄せ）、アイコン/見出し/行の文字サイズを拡大（56→64 / 18→24 / 13→14px）。s-tpl3・s-saas4 共通。

### Fixed
- **中断ラベルの文言是正**：一時保存ドラフトの `label` を「✓ 必須項目すべて完了(登録前準備完了)」→ **「必須ファイルをすべてアップ済み（本取り込み前）」**に。打刻記録の再開バナーで「完了」と誤読される問題を解消（owner-mock 側バナーと整合）。
- **「初期セットアップへ戻る」ボタンの遷移を確実化**（`bGoBackToSetupGuide`）：親タブ(初期セットアップ)から開かれた場合は `window.opener`（同一オリジン）で親を `#b0` に切り替えて前面化＋自タブを閉じる。単独タブ時はこの画面を `#b0` へ遷移。旧実装は `window.close()` 依存で単独タブ時に不発だった。
- **未アップロード時は「一時保存」「一時保存して中断」を押下不可に**（テンプレ／外部勤怠サービス両ウィザード s-tpl1 / s-saas1）。
  - 判定を新ウィザードのアップロード有無（`wizHasAnyUpload`）に一本化。`syncWizSaveButtons()` を `renderTplWizard()`/`renderSaasWizard()` 末尾で呼び、フッターの一時保存ボタン＋page-header の中断ボタンを同期。
  - 旧 `refreshSaveButtonsState` は legacy チェックボックス（ウィザードでは非表示）で判定しており、フッターだけ有効化される不整合があったのを是正。1件でもアップロードすると両ボタン有効化。
